# Exercício 01 — Commits Semânticos

Foram utilizados commits seguindo o padrão Conventional Commits.

## Tipos utilizados

- feat: novas funcionalidades
- fix: correção de erros
- refactor: melhorias sem alteração de comportamento
- docs: documentação

## Exemplos

- feat(calculadora): adiciona operações básicas
- fix(divisao): trata divisão por zero
- refactor(calculadora): melhora organização do código
- docs(readme): adiciona documentação do projeto

Os commits foram feitos de forma incremental, representando pequenas mudanças no sistema.
